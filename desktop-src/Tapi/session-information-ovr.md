# Session Information

TAPI applications must gather and process a variety of session information to function properly. For example, an answer operation fails unless the session is in the "offering" state.

**TAPI 2.x:** Applications primarily use the [**lineGetCallInfo**](https://msdn.microsoft.com/library/ms735720(v=VS.85).aspx) function to access session information. Additional functions which manipulate session information are [**lineGetCallStatus**](https://msdn.microsoft.com/library/ms735723(v=VS.85).aspx), [**lineSetCallParams**](https://msdn.microsoft.com/library/ms736086(v=VS.85).aspx), [**lineSetCallData**](https://msdn.microsoft.com/library/ms736084(v=VS.85).aspx), [**lineSetCallTreatment**](https://msdn.microsoft.com/library/ms736092(v=VS.85).aspx), [**lineSetCallPrivilege**](https://msdn.microsoft.com/library/ms736088(v=VS.85).aspx), [**lineGetCallStatus**](https://msdn.microsoft.com/library/ms735723(v=VS.85).aspx), and [**lineSendUserUserInfo**](https://msdn.microsoft.com/library/ms736067(v=VS.85).aspx).

**TAPI 3.x:** Applications primarily use the [**ITCallInfo**](/windows/desktop/api/tapi3if/nn-tapi3if-itcallinfo) interface to access session information.
